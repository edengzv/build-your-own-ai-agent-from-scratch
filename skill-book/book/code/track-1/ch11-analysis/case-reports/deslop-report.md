# 案例分析报告: deslop

> 本报告使用 7 维度框架分析 deslop Skill，展示完整的分析方法论。

## 基本信息

- **Skill 名称**: deslop
- **来源**: 社区（Anthropic Cookbook 衍生）
- **功能**: 消除 AI 生成文本中的"slop"（过度热情、不自然的语言模式）
- **行数**: ~80 lines
- **架构**: 单文件

---

## 维度 1: 触发设计

### Description 分析

| 检查项 | 结果 |
|--------|------|
| What | 消除 AI 生成文本中的 slop 模式 |
| When | 用户请求编辑文本、移除 AI 痕迹、改善写作风格 |
| Keywords | deslop, AI writing, slop, polish |
| 长度 | ~60 字 |
| 策略 | 适度 pushy |

### 评估

- 正触发率: **中**（"deslop"不是自然语言，多数用户不会这样说）
- 主要风险: 欠触发（用户更可能说"帮我改改这段文字"）
- **改进建议**: 在 When 中添加"用户说文字听起来像 AI 写的"、"请求修改语气"

**评分: 3/5**

### 核心洞察

1. **最大优势**: 约束设计精准——slop 模式列表把"模糊的美学标准"转化为"可检测的具体模式"
2. **最大弱点**: 缺少质量防线（无 Guardrails、无测试）
3. **最值得借鉴的设计决策**: 用"反模式列表"代替"正面描述"来定义质量标准
4. **最应该避免的做法**: Description 使用术语（"deslop"）而非用户自然语言

### 改进优先级

1. 添加 Guardrails: "不改变原文的事实内容和逻辑"
2. 扩展 Description 的 When 子句覆盖自然语言触发
3. 添加质量标准段
