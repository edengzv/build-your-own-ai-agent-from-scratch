# 部署检查清单

> 本文件供 Agent 在执行部署预检时查阅，确保覆盖所有关键检查点。

## 核心检查项

### 1. 环境变量

| 检查 | 阻塞级别 | 说明 |
|------|---------|------|
| 必需变量已设置 | 硬阻塞 | DATABASE_URL, API_KEY 等 |
| .env 不在版本控制中 | 硬阻塞 | .gitignore 必须包含 .env |
| .env.example 存在 | 警告 | 方便新成员配置 |
| 敏感变量加密存储 | 警告 | 使用 vault 或环境注入 |

### 2. 依赖

| 检查 | 阻塞级别 | 说明 |
|------|---------|------|
| Lock 文件存在 | 硬阻塞 | package-lock.json / poetry.lock |
| 无已知安全漏洞 | 警告 | npm audit / safety check |
| 依赖版本固定 | 警告 | 避免 ^ 或 ~ 前缀 |

### 3. 测试

| 检查 | 阻塞级别 | 说明 |
|------|---------|------|
| 测试全部通过 | 软阻塞 | 用户可决定是否继续 |
| 覆盖率 > 60% | 警告 | 基本覆盖 |
| 无跳过的测试 | 警告 | .skip / @pytest.mark.skip |

### 4. 构建

| 检查 | 阻塞级别 | 说明 |
|------|---------|------|
| 构建成功 | 硬阻塞 | 构建失败不可部署 |
| 无 TypeScript 错误 | 硬阻塞 | tsc --noEmit |
| 产物大小合理 | 警告 | 前端 bundle < 5MB |

### 5. 健康检查

| 检查 | 阻塞级别 | 说明 |
|------|---------|------|
| /health 返回 200 | 软阻塞 | Staging 环境 |
| 响应时间 < 5s | 警告 | 冷启动可能较慢 |
| 数据库连接正常 | 软阻塞 | /health/db |

## 阻塞级别定义

- **硬阻塞**：必须修复，不可继续部署
- **软阻塞**：展示给用户，由用户决定是否继续
- **警告**：记录在报告中，不阻止部署
