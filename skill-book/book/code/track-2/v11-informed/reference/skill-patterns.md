# Skill 设计模式参考

> 本文件供 Agent 在生成 Skill 时通过 Read 工具查阅，用于选择合适的结构模式。

## 模式一览

| 模式 | 适用场景 | 复杂度 | 典型行数 |
|------|---------|--------|---------|
| 单步模式 | 单一任务，输入→输出 | 低 | 10-50 |
| Multi-Pass 模式 | 需要收集→生成→验证 | 中 | 50-150 |
| 命令路由模式 | 多个独立子功能 | 中-高 | 100-200 |
| 三层架构模式 | 包含确定性逻辑+领域知识 | 高 | 150-300+ |

## 模式选择决策树

```
任务有几个独立子功能？
├── 1 个
│   ├── 需要几步？
│   │   ├── 1-2 步 → 单步模式
│   │   └── 3+ 步 → Multi-Pass 模式
│   └── 包含确定性逻辑？
│       ├── 是 → 三层架构模式
│       └── 否 → 保持当前模式
└── 2+ 个 → 命令路由模式
    └── 各子功能是否共享 Pass？
        ├── 是 → 命令路由 + 共享 Multi-Pass
        └── 否 → 命令路由 + 独立流程
```

## 详细模式说明

### 单步模式
适用于：quick-fix、commit-message 等简单任务。

### Multi-Pass 模式
关键原则：每个 Pass 有输入/输出产物，Pass 之间有暂停点。

### 命令路由模式
命名规范：动词优先（create/review/improve），正交设计。

### 三层架构模式
分层原则：scripts/（确定性）、reference/（知识）、SKILL.md（路由）。

### 工具编排模式
- Pipeline：工具按顺序串联
- Read-Analyze：多源收集后综合分析
- Conditional Branching：根据中间结果分支
